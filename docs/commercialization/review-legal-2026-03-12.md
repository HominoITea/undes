## 1. Лицензия Apache 2.0

**Вердикт: правильный выбор.** Менять не нужно.

Правовая основа: Закон РК "Об авторском праве и смежных правах" (No 6-I), ст. 16, 30, 31, 32; ГК РК (Особенная часть), ст. 964-972.

Apache 2.0 -- неисключительный лицензионный договор (ст. 32 Закона), допустимый в РК. Patent grant ценен для B2B. Copyleft отсутствует -- совместимо с open-core.

**Подводные камни:**
- Нет привязки к юрисдикции. Рекомендация: добавить в README/LEGAL NOTICE: "В случае споров применяется законодательство Республики Казахстан."
- Disclaimer of warranty (раздел 7 Apache 2.0): по ГК РК (ст. 350, 359) полное исключение ответственности может быть ограничено. В B2B договоре на pilot нужен отдельный раздел limitation of liability.
- AI-генерированный код: Apache 2.0 требует, чтобы Licensor имел право лицензировать. Вопрос авторства AI-кода -- см. пункт 2.

## 2. IP / Авторские права

Правовая основа: Закон об авторском праве, ст. 6, 9, 14, 15-16; ГК РК (Общая часть), ст. 971.

**Код founder'а:** Авторское право возникает в момент создания (ст. 6). Git history -- доказательство. Если founder регистрирует ТОО -- нужен **договор уступки имущественных прав** (ст. 30) от физлица к ТОО.

**AI-генерированный код:** Нет прямой нормы РК. Закон (ст. 9) определяет автором физическое лицо. Практический подход: код, сгенерированный AI по промпту человека = произведение с использованием технического средства. Автор -- лицо, осуществлявшее творческий вклад.

**Документальная фиксация:** В CONTRIBUTING.md: "Код, созданный с использованием AI-инструментов, проверен, отредактирован и принят founder'ом."

**Риск:** Суд РК может признать чисто AI-генерированный код не объектом авторского права (= public domain). Для OSS это не проблема, но для paid layer -- потенциально.

**Необходимые документы:**

| IP Assignment: founder → ТОО | Высокий | До публикации |

| CONTRIBUTING.md с DCO | Высокий | До публикации |
| Оговорка об AI-генерированном коде | Средний | До публикации |

## 3. Paid Pilot -- договорная конструкция

Правовая основа: ГК РК (Особенная часть), глава 33, ст. 683-686; Предпринимательский кодекс, ст. 30-35.

**Рекомендация: ТОО** (не ИП). Причины: ограничение ответственности, IP на юрлицо, B2B стандарт, возможность привлечения инвесторов.

**Структура договора:**
1. Предмет: услуги по проведению пилотного проекта (ст. 683 ГК)
2. SOW как приложение: 1 репо, 1 команда, 8-10 задач, 14 дней, readout
3. Стоимость: $3,000 (эквивалент в тенге по курсу НБ РК), 70% предоплата, валютная оговорка
4. Change Request: отдельная процедура ($1,000+)
5. NDA: раздел или отдельный документ
6. IP allocation: OSS core под Apache 2.0; кастомизации -- определить владельца; AI-артефакты -- оговорить
7. Limitation of liability: ограничить суммой договора, исключить consequential damages
8. Споры: законодательство РК, медиация → суд (или AIFC Court для международных)

## 4. Налоги

Правовая основа: НК РК (No 120-VII).

**Оптимальный режим:** ТОО на упрощённой декларации (ст. 683-686 НК). Ставка 3% от дохода. Лимит: до 24,038 МРП за полугодие.

**Cross-border:**
- Экспорт IT-услуг: НДС 0% (ст. 370, 441 НК)
- Валютная выручка: открыть валютный счёт, порог учётной регистрации $50,000
- Withholding tax при оплате API нерезидентам: потенциально 15-20% КПН у источника (ст. 644-655 НК). Проверить конвенцию с США.

## 7. Непокрытые риски

**Высокий приоритет:**
1. **Нет юрлица.** Без ТОО/ИП невозможно легально принять оплату. Регистрация 1-3 дня через eGov.
2. **AI-код и IP.** Провести IP audit, зафиксировать AI-generated portions.
3. **Ответственность за AI-результат.** В договоре: исключить гарантию безошибочности, ограничить liability суммой договора, обязать клиента проверять результаты.
4. **Export control.** Проверить GitHub ToS, криптографические компоненты.

**Средний приоритет:**
5. Валютное регулирование (порог $50,000).
6. Terms of Service OpenAI/Anthropic -- совместимость с коммерческим использованием.
7. Явная оговорка отсутствия SLA в договоре.
8. `docs/commercialization/` и `COMMERCIAL_ADDON_BOUNDARY.md` не должны попасть в публичный репо.
